# Internationalization (i18n)

The Volcano dashboard uses [next-intl](https://next-intl-docs.vercel.app/) with locale-prefixed routes (`/en/...`, `/zh-CN/...`).

## Folder layout

```
apps/web/
  messages/
    en/           # Source locale (schema of truth)
      common.json
      nav.json
      dashboard.json
      jobs.json
      queues.json
      pods.json
      podgroups.json
    zh-CN/        # Mirror structure for each locale
  src/i18n/
    config.ts     # locales, defaultLocale, direction helpers
    routing.ts    # defineRouting for middleware + navigation
    request.ts    # getRequestConfig (loads messages)
    navigation.ts # locale-aware Link, useRouter, usePathname
  src/middleware.ts
```

## Key naming

Use nested namespaces matching JSON files: `jobs.table.name`, `common.actions.delete`.

Use ICU syntax for plurals and interpolation:

```json
"jobsRunningDescription": "{count, plural, one {# job is running} other {# jobs are running}}"
```

## Using translations

**Client components:**

```tsx
import { useTranslations, useFormatter } from "next-intl";

const t = useTranslations("jobs");
const format = useFormatter();

t("table.name");
format.dateTime(date, { dateStyle: "medium", timeStyle: "short" });
```

**Server components / metadata:**

```tsx
import { getTranslations } from "next-intl/server";

const t = await getTranslations({ locale, namespace: "dashboard" });
```

**Navigation:** Prefer `usePathname` from `@/i18n/navigation` (locale stripped). Use `NextLink` with `/${locale}${href}` or helpers from `@/i18n/navigation` where types allow.

## Adding a new locale

1. Copy `messages/en/` to `messages/<locale>/` and translate all values.
2. Add the locale to `src/i18n/config.ts` (`locales`, `localeLabels`).
3. Add a label in the language switcher (`localeLabels` in `config.ts`).
4. Run `npm run i18n:validate` from `apps/web`.
5. Smoke-test all routes under `/<locale>/`.

## What not to translate

- Kubernetes / Volcano API field names in YAML editors
- Raw cluster status strings when shown as technical values
- Resource kind names (`Job`, `Queue`, `Pod`) in manifests

## Validation

From `apps/web`:

```bash
npm run i18n:validate
```

Compares key paths in every locale against `en`.

## Future: translation platforms

For larger teams, export `messages/en/**/*.json` to Crowdin or Lokalise and import completed locales into `messages/<locale>/`.
